## 6. User interface prototype

이 장은 예상 UI 및 UI 안의 각 구성요소를 설명한다. 최종적으로 개발될 디자인의 prototype이며, 세부적인 사항은 달라질 수 있다.

- 하단 내비바의 대시보드는 마이페이지로 이름을 변경한다.

### 6.1. 로그인

![[그림 6-1] 로그인 페이지](./image/6-1.png)
[그림 6-1] 로그인 페이지

 [그림 6-1]은 서비스 초기 진입 화면인 로그인 페이지이다. 사용자는 등록된 ID(이메일)와 비밀번호를 입력하여 서비스에 접근한다. 또한, 계정이 없는 비회원을 위해 회원가입 페이지로 이동하는 버튼을 제공한다.

### 6.2. 회원가입

![[그림 6-2] 회원가입 페이지1](./image/6-2.png)
[그림 6-2] 회원가입 페이지1

![[그림 6-3] 회원가입 페이지2](./image/6-3.png)
[그림 6-3] 회원가입 페이지2

 [그림 6-2]와 [그림 6-3]은 신규 사용자가 서비스 이용을 위해 계정을 생성하는 회원가입 페이지이다. 사용자는 양식에 따라 이름, 이메일(ID), 비밀번호, 전화번호, 전공/직무 등 필요한 정보를 입력한다.

### 6.3. 마이페이지

![[그림 6-4] 대시보드 페이지](./image/6-4.png)
[그림 6-4] 메인 페이지

![[그림 6-5] 프로젝트 생성 시](./image/6-5.png)
[그림 6-5] 프로젝트 생성 시

 [그림 6-4]는 로그인 성공 시 가장 먼저 보게 되는 메인 페이지이다. 왼쪽의 사이드바를 통해 사용자가 참여 중인 프로젝트의 목록을 확인할 수 있다. 메인 페이지는 사용자가 참가자로 등록된 모든 프로젝트의 일정을 통합하여 달력에 표시한다. 일정은 해당 날짜에 프로젝트명과 일정 이름으로 출력된다. 

 페이지의 상단 내비바에는 다른 사용자가 생성한 프로젝트에 참여할 수 있도록 참여코드를 입력할 수 있는 검색과 프로젝트를 생성할 수 있는 버튼이 있다. 이때, 프로젝트 생성 버튼을 누를 경우, [그림 6-5]와 같이 사이드바에 새 프로젝트가 만들어진다. 또한, 생성 버튼 옆을 보면 사용자의 이름을 확인할 수 있고, 로그아웃, 설정 버튼이 각각 존재한다.

### 6.5. 프로젝트 관리

![[그림 6-6] 프로젝트 관리 페이지 (관리자)](./image/6-6.png)
[그림 6-6] 프로젝트 관리 페이지 (관리자)

![[그림 6-7] 프로젝트 관리 페이지 (참여자)](./image/6-7.png)
[그림 6-7] 프로젝트 관리 페이지 (참여자)

 이 페이지는 개별 프로젝트의 전반적인 설정을 관리하는 화면으로, 하단 내비바의 프로젝트 관리를 누를 경우 이동된다. [그림 6-6]는 프로젝트 관리자 권한을 가진 사용자의 화면이다. 프로젝트명을 수정하고, 프로젝트 멤버의 참여 요청을 승인 또는 거절하거나, 기존 멤버를 삭제하거나 관리자 권한을 양도하는 등 멤버 관리가 가능하다. 또한, 프로젝트의 참여코드를 확인 및 복사할 수 있으며 프로젝트 삭제가 가능하다.
 
 [그림 6-7]은 일반 참여자 권한을 가진 사용자의 화면이다. 프로젝트 참여코드와 프로젝트 멤버만 확인할 수 있다. 또한, 프로젝트 삭제가 아닌 나가기가 가능하다.
 
![[그림 6-8] 프로젝트 삭제](./image/6-8.png)
[그림 6-8] 프로젝트 삭제

![[그림 6-9] 프로젝트 나가기](./image/6-9.png)
[그림 6-9] 프로젝트 나가기

 [그림 6-8]는 관리자가 프로젝트를 영구적으로 삭제하는 인터페이스이다. 삭제하기 위해서는 프로젝트명을 입력해야 한다. [그림 6-9]은 일반 참여자가 해당 프로젝트에서 나가는 인터페이스로, 영구적으로 삭제되지 않는다. 프로젝트 삭제와 동일하게, 나가기 위해서는 프로젝트명을 입력해야 한다.

[그림 6-10] 프로젝트 페이지

![[그림 6-12] 일정 추가 및 수정](./image/6-12.png)
[그림 6-12] 일정 추가 및 수정

![[그림 6-13] 일정 조회](./image/6-13.png)
[그림 6-13] 일정 조회

 프로젝트의 일정을 월별 캘린더 형식으로 관리하는 페이지로, 메인 페이지에서 프로젝트를 선택하거나, 하단 내비바의 마이페이지를 누를 경우 [그림 6-10]으로 이동된다. [그림 6-11]은 사용자가 특정 날짜를 선택하면, 사이드바에 해당 날짜에 등록된 일정 목록이 나타나 확인할 수 있다. [그림 6-12]는 일정 추가 버튼을 눌렀을 때의 화면으로, 일정 명, 기간, 내용, 참가자 등 상세 정보를 입력하여 새 일정을 생성한다. 또한, 기존 일정의 수정 아이콘을 눌러 내용을 편집할 수 있고, 삭제 아이콘을 눌러 일정을 삭제할 수 있다. [그림 6-13]은 [그림 6-11]의 사이드바의 일정 목록에서 특정 일정을 클릭했을 때, 해당 일정의 상세 정보를 모달창으로 보여준다.
 
### 6.7. 게시판 페이지

![[그림 6-14] 게시판 페이지](./image/6-14.png)
[그림 6-14] 게시판 페이지

![[그림 6-15] 게시글 작성 페이지](./image/6-15.png)
[그림 6-15] 게시글 작성 페이지

![[그림 6-16] 투표 생성](./image/6-16.png)
[그림 6-16] 투표 생성

![[그림 6-17] 작성한 게시글 페이지](./image/6-17.png)
[그림 6-17] 작성한 게시글 페이지

 프로젝트 참여자 간의 의사소통을 위한 게시판으로 하단 내비바의 게시판을 누를 경우 [그림 6-14]의 게시판 페이지로 이동된다. 게시글은 기본적으로 작성 순으로 내림차순 정렬되며, 공지사항으로 설정된 게시글은 상단에 고정되어 표시된다. 이때 공지사항의 제목은 볼드체, 글 번호는 아이콘으로 표시한다. 게시판의 목록에서는 글 번호, 제목, 작성자, 작성일자, 투표와 첨부 파일의 존재 여부를 알 수 있다.

 [그림 6-15]는 [그림 6-14]에서 게시글 작성 버튼을 눌러 이동되는 새 글을 작성하는 페이지이다. 제목과 내용을 입력하고, 파일을 첨부하거나 공지사항의 유무를 체크할 수 있으며, 투표 추가할 수 있다. [그림 6-16]은 게시글 작성 시 투표 생성 버튼을 눌렀을 때 나타나는 모달창이다. 투표 항목과 마감일, 중복 선택이나 익명 투표의 여부를 설정한다.

 [그림 6-17]은 작성이 완료된 게시글을 확인할 수 있는 페이지이다. 제목, 본문 내용 등 게시글에 대한 정보를 알 수 있고, 첨부 파일과 생성된 투표가 존재한다면 확인할 수 있으며, 투표가 마감되지 않았다면 투표에 참여할 수 있다. 추가적으로 게시글의 작성자인 경우 수정과 삭제 버튼도 존재한다.

[그림 6-18] 설정 페이지

![[그림 6-19] 정보 변경](./image/6-19.png)

![[그림 6-20] 비밀번호 변경](./image/6-20.png)

 사용자 본인의 개인정보를 관리하는 페이지로, [그림 6-18]은 상단 내비바의 설정을 누르고 들어갔을 때 표시된다. 해당 페이지에서 사용자의 이름, 이메일, 전화번호, 전공/직무 등 개인정보를 확인할 수 있다. 수정하기 버튼을 누를 경우, 이름, 비밀번호, 전화번호, 전공/직무를 수정할 수 있는데, [그림 6-19]는 예시로 전화번호를 변경한 상황이다. 이름이나 전공/직무도 동일하게, 변경하고자 하는 내용으로 입력하면 변경할 수 있다. 비밀번호 변경은 [그림 6-20]과 같이 현재 비밀번호, 새 비밀번호, 새 비밀번호 확인 총 3가지를 입력해야 한다.

![[그림 6-21] 회원 탈퇴](./image/6-21.png)
[그림 6-21] 회원 탈퇴

 또한, 회원 탈퇴 버튼도 [그림 6-18]에서 볼 수 있듯이 설정 페이지에서 제공된다. 회원 탈퇴 버튼을 누르면 [그림 6-21]과 같은 회원 탈퇴 창이 나타난다. 회원 탈퇴를 위해서는 사용자의 비밀번호를 입력해야 한다.

### 6.9. 시간조율

![[그림 6-22] 시간조율표 생성 및 리스트](./image/6-22.png)
[그림 6-22] 시간조율표 생성 및 리스트

![[그림 6-23] 시간조율표 편집](./image/6-23.png)
[그림 6-23] 시간조율표 편집

 하단 내비바의 시간조율을 누를 경우 이동되며, 팀원들 간의 공통 가능 시간을 쉽게 찾을 수 있도록 돕는 기능 페이지이다. [그림 6-22]는 새 시간조율표를 생성하는 인터페이스와 이미 생성된 시간조율표의 리스트를 보여준다. 시간조율표가 존재한다면 [그림 6-23]과 같은 시간조율표 편집 화면을 확인할 수 있다. 왼쪽 표에는 사용자 개인의 가능 시간을 드래그하여 표시하는 영역이며, 오른쪽 표는 해당 시간 조율표를 편집한 사용자 전원의 가능 시간이 합산되어 나타나는 결과이다. 이때, 겹치는 인원수에 따라 색의 투명도가 다르게 표시된다.
 
### 6.10. 사이드바

![[그림 6-24] 사이드바](./image/6-24.png)
[그림 6-24] 사이드바

 [그림 6-24]는 서비스의 모든 페이지에서 공통으로 제공되는 사이드바이다. 하단 내비바의 프로젝트 목록을 누르면 나타나며, 사용자가 참여 중인 프로젝트 목록을 확인할 수 있다. 또한, 다른 프로젝트를 선택하여 이동할 수 있다.
