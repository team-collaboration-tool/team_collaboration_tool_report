# 5. State machine diagram

 본 장은 '협업의 민족' 시스템의 사용자 인터페이스(UI) 동작 흐름을 시각적으로 모델링한 State machine diagram(이하 SMD)에 대해 기술한다. [그림 5-1]은 시스템의 주요 기능을 수행하는 화면 간의 전이 관계를 프로젝트 관리자의 관점에서 도식화한 것이다. 본 SMD는 시스템 내부 데이터의 상태 변화보다는 사용자와 시스템 간의 상호작용에 따른 화면 내비게이션 흐름을 표현하는 데 중점을 두었다.

### 5.1. 상태 정의
 
시스템의 주요 상태와 각 상태에서 수행되는 기능은 다음과 같다.
- 로그인 : 시스템 접속 시 최초로 진입하는 상태로, 사용자 인증을 대기한다. 로그인 성공 시 세션이 생성되며 마이페이지로 전이된다.
- 마이페이지 : 인증된 사용자의 개인화 홈 화면이다. 개인 일정 확인, 프로젝트 목록 조회, 개인정보 설정 진입이 가능하다.
- 설정 : 개인정보 수정 및 회원 탈퇴 기능을 수행할 수 있다.
- 프로젝트 : 협업 활동이 이루어지는 상위 상태로, 내부에 달력, 게시판, 시간조율, 프로젝트 관리 기능을 포함한다.
  - 달력 : 프로젝트 진입 시 기본 화면이다. 일정 조회, 추가, 수정, 삭제 활동을 수행한다.
  - 게시판 : 팀원 간 소통을 위한 게시글 작성, 수정, 삭제 활동을 수행한다.
  - 시간조율 : 회의 시간 등을 정하기 위한 시간조율표 생성 및 편집 활동을 수행한다.
  - 프로젝트 관리 : 프로젝트명 수정, 멤버 관리 등을 수행한다.
- 종료 : 사용자가 로그아웃을 요청하여 세션이 만료되고 시스템 사용이 종료된 상태이다.
 
### 5.2. 전이 및 이벤트

- 기본 흐름
  - 로그인 → 마이페이지 : 사용자가 올바른 아이디와 비밀번호를 입력하고 로그인 버튼을 클릭하면 전이된다.
  - 전체 상태 → 종료 : 마이페이지, 설정, 프로젝트 전체 중 어느 상태에서든 사용자가 로그아웃 버튼을 클릭하면 즉시 세션을 종료하고 시스템을 빠져나간다.

- 프로젝트 진입 및 전환
  - 마이페이지 → 프로젝트 : 프로젝트 목록에서 특정 프로젝틀르 선택하면 해당 프로젝트의 작업 공간으로 진입한다.
  - 프로젝트 → 마이페이지 : 하단 내비바의 마이페이지 버튼을 클릭하면 프로젝트 작업을 중단하고 개인화 홈 화면으로 복귀한다.
  - 현재 프로젝트 → 다른 프로젝트 : 프로젝트 수행 중 사이드바를 통해 다른 프로젝트를 선택하면 해당 프로젝트의 작업 공간으로 진입한다.

- 내비바
  - 상단 내비바 : 시스템 전역에서 접근 가능하며, 새 프로젝트를 생성하거나 코드를 통해 다른 프로젝트에 참여할 수 있다. 또한, 설정 화면으로 진입하거나 로그아웃을 통해 시스템을 종료하는 흐름을 담당한다.
  - 하단 내비바 : 특정 프로젝트에 진입했을 때 활성화되며, 프로젝트 내부 기능인 달력, 시간조율, 게시판, 프로젝트 관리의 4가지 복합 상태 간의 수평적 전이를 담당한다.
